<h2>Repository Structure</h2>
<p>
This repository is organised into a clear, modular structure that separates raw data,
exploratory notebooks, and generated visualisations.
</p>

<ul>
  <li>
    <strong>Data/Raw/</strong><br>
    This directory is intentionally empty in the repository. The official datasets are
    too large to upload, so each contributor downloads the data locally and places the
    files here before running any analysis.
  </li>

  <li>
    <strong>Notebooks/</strong><br>
    Contains the main Jupyter notebooks used for the analyses. Each team member has a
    dedicated notebook:
    <ul>
      <li><strong>InterSubjects_Correlations.ipynb</strong> – compares subjects, studies correlations
          and statistical dependencies.</li>
      <li><strong>BirthMonth.ipynb</strong> – analyses how grades vary depending
          on month of birth and other temporal patterns.</li>
      <li><strong>timeEvol.ipynb</strong> – examines the evolution of grades over
          the years.</li>
      <li><strong>exploration-peter.ipynb</strong> – focuses on demographic and geographic
          aspects of performance.</li>
    </ul>
  </li>

  <li>
    <strong>InterSubjects_CorrelationsPlots/</strong>,
    <strong>BirthMonthPlots/</strong>,
    <strong>AdrianPlots/</strong>,
    <strong>PeterPlots/</strong>,
    These folders store figures and intermediate outputs generated by the notebooks,
    grouped by method or topic (e.g. scatter plots, joint distributions, density
    difference heatmaps, randomized-baseline explorations).
  </li>
</ul>

<p>
Together, this layout keeps the project modular and reproducible while making it easy to
identify where data should be placed and how each team member’s analysis contributes to
the overall study.
</p>
